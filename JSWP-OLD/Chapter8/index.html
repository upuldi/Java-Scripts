Java Scripts executes codes synchronizely. In an JS engine there is an event queue which used to keep events queued and once the execution context finished executed the JS engine beging to process those events. When it processing those events it creates an execution context for thsoe. For ex an event might be registed to a function. Browser put events into the event queue when the user is working on the browser.

JS engine always process the event queue after processing the execution stack.
